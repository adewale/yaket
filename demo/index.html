YAKE in the browser

See what makes Yaket different in under a minute.

Yaket is a JavaScript port of YAKE: a single-document, collection-independent keyword extraction approach that uses local statistical features instead of embeddings, training data, or external corpora.

Why YAKE feels different

  • Single-document: it works from the text in front of it, not a training set.
  • No training corpus: no embeddings, no vector store, no model checkpoint.
  • Local statistical features: frequency, position, spread, casing, and co-occurrence drive the ranking.

How to read the output

Lower score means more relevant. Each result also shows occurrences and sentence spread so you can judge whether a phrase is specific, repeated, or broadly distributed.

Fast enough for client-side use

Runs directly in the browser via the published npm package. Useful for browser extensions, docs tooling, and no-backend tagging flows.

Closer to upstream YAKE

Yaket tracks the Python YAKE pipeline and publishes parity benchmarks, rather than just shipping a vague “keyword-ish” heuristic.

Good fit for indexing and tagging

Works well for article metadata, search indexing, newsletter topics, and RAG chunk enrichment when you want a deterministic non-LLM baseline.

    What this demo proves

    Against TF-IDF

    YAKE uses richer local features than raw term counts, so it can surface phrases that feel more document-specific.

    Against RAKE

    YAKE is less dependent on stopword chunking and uses a more nuanced scoring pipeline.

    Against KeyBERT

    YAKE avoids embedding-model cost and works as a deterministic, lightweight baseline that can run anywhere.
